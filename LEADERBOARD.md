## Methodology

- **42 projects** across 30+ languages/frameworks
- **~4,200 planted issues** (security, logic, performance, best practices, code smells, tricky cross-module bugs)
- **~300 red herrings** (code that looks vulnerable but is safe)
- **Scoring:** Detection (1pt) + Severity (0.5) + CWE (0.5) + Location (0.5) + Fix (1.0) + Explanation (0.5)
- **Difficulty multipliers:** Tier 1 (×1.0) → Tier 5 (×3.0)
- **Category weights:** SEC 35% | TRICKY 25% | LOGIC 20% | PERF 10% | BP 5% | SMELL 5%
- **Penalties:** -1.0 per false positive, -2.0 per flagged red herring
- **Ground truth** encrypted with AES-256-GCM, unavailable to reviewers during testing
