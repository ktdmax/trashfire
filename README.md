## What is this?

TRASHFIRE is a benchmark for evaluating AI code review tools. Each project is a realistic mini-application with 100 carefully planted issues  - from obvious SQL injections to subtle cross-module race conditions that even senior developers would miss.

**The rules are simple:** Give an AI the code. See what it finds. Score it against encrypted ground truth.

## Issue Categories

Not just security bugs. We test **six dimensions** of code review ability:

| Category | Weight | What it tests |
|----------|-------:|---------------|
| **SEC**  - Security | 35% | OWASP Top 10, CWE-mapped vulnerabilities |
| **TRICKY**  - Cross-module | 25% | Bugs that only manifest across function chains, edge cases, timing |
| **LOGIC**  - Logic bugs | 20% | Off-by-one, wrong operators, race conditions, async issues |
| **PERF**  - Performance | 10% | N+1 queries, memory leaks, blocking I/O, ReDoS |
| **BP**  - Best practices | 5% | Hardcoded values, swallowed errors, deprecated patterns |
| **SMELL**  - Code smells | 5% | Dead code, duplication, god functions |

### Difficulty Tiers

| Tier | Multiplier | Example |
|------|-----------|---------|
| 1  - Obvious | ×1.0 | Hardcoded `password = "admin123"` |
| 2  - Standard | ×1.25 | SQL injection in query builder |
| 3  - Moderate | ×1.5 | SSRF with partial validation bypass |
| 4  - Hard | ×2.0 | Cross-module race condition |
| 5  - Expert | ×3.0 | Multi-file timing attack + type coercion chain |

Finding a Tier 5 bug is worth **3× more** than finding a Tier 1 bug.

### Red Herrings

Each project includes 5-10 **red herrings**  - code that *looks* vulnerable but is actually safe. Flagging these costs you **-2.0 points** (double the penalty of a regular false positive). Examples:

- `innerHTML` after `DOMPurify.sanitize()` (sanitized 3 lines above)
- `eval()` on a `JSON.parse()` result (already safe)
- MD5 used for cache ETags (not for passwords)
- SQL string concat with a hardcoded constant (not user input)

## Quick Start

```bash
# 1. Clone and install
git clone https://github.com/ktdmax/trashfire.git
cd trashfire
cd _scoring && npm install && cd ..

# 2. Generate the blind testing copy (strips all bug markers)
bash _scoring/create-blind-copy.sh

# 3. Run a benchmark (interactive - picks project + prompt for you)
bash run-benchmark.sh

# 4. Or specify project and preset directly
bash run-benchmark.sh grog-shop              # uses default prompt
bash run-benchmark.sh tentacle-labs "Your custom review prompt here"
```

## Scoring

Each correctly identified issue scores up to **4.0 raw points**, multiplied by difficulty:

| Component | Points |
|-----------|--------|
| Found it | 1.0 |
| Correct severity | +0.5 |
| Correct CWE | +0.5 |
| Exact file + line | +0.5 |
| Working fix | +1.0 |
| Clear explanation | +0.5 |

**Penalties:** -1.0 per false positive, -2.0 per flagged red herring.

## Anti-Cheating

- Ground truth manifests are **AES-256-GCM encrypted** with a passphrase only provided at scoring time
- All `// BUG-XXXX:` markers are **stripped** from the blind testing copy
- Git history is squashed (no manifest edits visible)
- Snippet hashes in manifests detect code tampering
